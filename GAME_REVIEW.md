## Overall system assessment

### The complete action → feedback chain

**Shortlist a candidate:**
1. Tick fires → `game:shortlist_added` event
2. Score pop: ⭐ `+50 XP` (immediate)
3. Daily objective counter increments (e.g. "Shortlist 3 candidates today: 1/3")
4. `shortlistCount` query invalidated → refetches within ~1s
5. `useGameStats` recalculates with new count → XP total updates
6. Achievement check (e.g. "Selective Eye" if first shortlist) → possible 🏅 pop
7. XP delta detected → possible generic "+N XP" pop
8. Level change detected → possible `LevelUpOverlay`

**Post a role:**
1. Role saved → `game:role_created` event
2. Score pop: 🎯 `+100 XP` (immediate)
3. Daily objective counter increments
4. `useRoles` refetch → stats update with new role count

**Make an introduction (AI-detected):**
1. AI pipeline sets `_matches.status = 'introduced'`
2. Next `useRoles` refetch (staleTime 2min) → `stats.intros` increases
3. `prevIntrosRef` detects delta → fires `🤝 +200 XP` pop per new intro
4. `game:intro_made` event fires → daily intro objective increments
5. Achievement check + possible level-up

### What makes a good gamification loop

The three essentials for a gamification loop are:
1. **Clear action** — recruiter knows what to do
2. **Immediate feedback** — something happens NOW when they do it
3. **Progress tracking** — they can see how far they've come

DashTrees now has all three for shortlisting and role-posting. Intros work but with a delay (inherent to AI pipeline). The daily objectives give fresh daily goals. The leaderboard provides social comparison. Level-up celebrations mark milestones.
