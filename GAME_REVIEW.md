### Why `candidate_added` XP is blocked

`_talent` rows are inserted without any recruiter attribution:
```typescript
// server/index.ts — /api/candidates/upload
await supabase.from('_talent').insert({ name: ..., lifecycle_state: 'imported' })
```
No `created_by` column exists in migrations. To fix:
1. Write migration: `alter table _talent add column if not exists created_by uuid references _recruiters(id) on delete set null`
2. Update `/api/candidates/upload` and `/api/candidates/import-text` to set `created_by: req.auth!.recruiterId`
3. Add `GET /api/candidates/count` → `count where created_by = recruiterId`
4. Wire into `useGameStats` → `candidatesXP = count * 75`
5. Fire `game:candidate_added` event from `NewCandidateScreen.onCreated`
