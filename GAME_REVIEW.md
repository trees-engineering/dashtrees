### Team leaderboard (added this iteration)

Server-side aggregation at `GET /api/game/leaderboard`:
- Fetches `_role`, `_matches`, `_shortlists`, `_recruiters` in parallel (4 queries)
- Groups roles by recruiter UUID, computes rolesXP + matchesXP + shortlistedXP + introsXP + achievementXP per person
- Achievement XP computed server-side with identical logic to `gamification.ts`
- Returns sorted descending by totalXP

`GamePlayerTab` wraps `ProfileTab` and prepends:
1. Own rank banner: rotating diamond badge, level title, XP total, "#N on team" indicator
2. Full team leaderboard with gold/silver/bronze medals, "YOU" tag on own row
3. Divider, then the existing ProfileTab form below

PlayerTab was previously a plain form — now it's a proper player profile page.

### Overall assessment after 6 iterations

The game mode is now substantially more gamified than when the loop started:

**Feedback loops working:** Shortlist → immediate +50 XP pop → daily objective progress → achievement check → possible level-up overlay. The chain from action to reward is complete for the core shortlisting flow.

**Missing feedback:** Intros still have no immediate pop. The recruiter makes an intro, nothing happens in the UI game-wise until the next `useRoles` refetch. Fixing this requires either: (a) a webhook/subscription on `_matches.status` changes, or (b) firing `game:intro_made` from `IntrosTab` when the recruiter manually marks a status as "introduced". Option (b) is easier.

**Candidate XP gap:** Still the main structural hole. Until `_talent.created_by` exists, CV uploads contribute 0 explicit XP and no pop. The score pop can at least be added cheaply — fire `game:candidate_added` from `NewCandidateScreen.onCreated` and handle it in GameDashboard. But the XP won't accumulate in the total.

**Visual polish:** Daily objectives look clean. Leaderboard is functional. The biggest visual gap remaining is the plain ProfileTab form visible below the rank banner — the form fields are unstyled relative to the game aesthetic.

_The core gamification loop is now solid. Next improvements are polish and the intro feedback gap._
