### Shortlist XP: what changed

Before this iteration:
- `useGameStats` read `r.counts.shortlisted` (count of `_matches.status = 'shortlisted'`)
- That field is set by the AI pipeline, not by recruiter actions
- Recruiters ticking candidates in the UI did nothing to their XP total

After this iteration:
- New server route `GET /api/shortlists/count` counts `_shortlists` rows for `req.auth!.recruiterId`
- `useGameStats` fetches this count (stale 30s) and passes it as `shortlisted` to `computeGameStats`
- Every tick-box toggle now affects XP total, achievements, stars, and quest progress
- `_matches.status = 'shortlisted'` is still used by the IntrosTab for display; game XP is now independent
